  • Hoisting: Variables declared with var are hoisted to the top of their
  • scope and initialized with undefined.
  • Re-declaration: You can re-declare the same variable multiple times without error.
  • Use case: Older JavaScript code predominantly uses var, but it has some quirks that can lead to bugs.

let Scope: Block-scoped (only accessible within the block {} where it is declared). Hoisting: Variables declared with let are hoisted but not initialized, causing a "temporal dead zone" until their declaration is reached. Re-declaration: You cannot re-declare a variable with let in the same scope. Use case: Preferred for variables that will change value and need block scoping.

const Scope: Block-scoped, like let. Hoisting: Same behavior as let with temporal dead zone. Re-declaration: Cannot be re-declared or reassigned. Use case: For variables that should never be reassigned (constants). However, if the variable holds an object or array, the contents can still be modified.
